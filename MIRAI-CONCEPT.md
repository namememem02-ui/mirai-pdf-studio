## 2. ภาพรวมระบบนิเวศ Mirai

- **Mee-a-rai Hub** (`Desktop\Mirai Gallery`) = ทำเนียบกลางธีม JRPG 16-bit รวม "ยูนิต" (โปรเจ็ค) ทั้งหมด จัดกลุ่มเป็น "กิลด์"
  - Hub ออนไลน์: https://mirai-guild-hub.vercel.app (โหมดอ่านอย่างเดียว) | แก้ไขที่ localhost:3000
  - ข้อมูลกิลด์ = ไฟล์ `data/guilds/<กิลด์>.json` (1 ไฟล์ต่อ 1 กิลด์ — ยูนิตผูกสัมพันธ์ได้เฉพาะในกิลด์เดียวกัน ข้ามกิลด์ไม่ได้)
  - กิลด์ "ผู้มาจากต่างโลก" = ที่อยู่ของไลบรารี/เครื่องมือที่**ไม่ได้สร้างเอง**
- **โปรเจ็คใหม่ทุกตัว** สร้างในโฟลเดอร์ `Desktop\Mirai HUB\<ชื่อโปรเจ็ค>` แยกเป็นเอกเทศ มี git ของตัวเอง
- **Workflow ประจำทีม**: ผู้ใช้คิดร่วมกับ Claude (วางแผน/รีวิว/ทดสอบ) → Gemini เป็นคนเขียนโค้ดหลัก → Claude ตรวจงานด้วยการรัน/ยิงทดสอบจริงเสมอ (ไม่เชื่อแค่ตาเปล่า) — สื่อสารภาษาไทย

## 4. กติกาเทคนิค (Convention)

1. **Stack มาตรฐาน**: Next.js 16.2.10 + React 19.2.4 + Tailwind v4 + TypeScript (เว้นแต่งานบังคับเป็นอย่างอื่น) — Next 16 มี breaking changes ให้อ่าน docs ใน `node_modules/next/dist/docs/` ก่อนเขียน, `params` เป็น Promise ต้อง `await params`
2. **UI สองแบบ** เลือกตามชนิดงาน:
   - เครื่องมือสำหรับคนทั่วไป → UI เรียบ สะอาด พื้นขาว (แบบ Mee-a-rai PDF Studio)
   - ส่วนของ Hub/ธีมเกม → JRPG 16-bit (FEWindow กรอบทอง ฟอนต์พิกเซล)
   - ภาษาไทยเป็นหลักเสมอ + ถ้ามีการวาดข้อความลง PDF/รูป ต้องใช้ฟอนต์ไทย (Sarabun ฝังในโปรเจ็ค ห้ามพึ่ง CDN)
3. **เว็บสาธารณะ = ประมวลผลฝั่งเบราว์เซอร์ (client-side) เป็นค่าเริ่มต้น** ไฟล์ผู้ใช้ห้ามขึ้น server — ถ้าจำเป็นต้องมี server จริงให้ปรึกษาผู้ใช้ก่อน
4. **ทุกโปรเจ็คต้องมี**: `start.bat` (รัน dev + เปิดเบราว์เซอร์), `README.md` (วิธีเปิดใช้ + วิธีเพิ่มฟีเจอร์), `.gitignore`
5. **Git**: `git init` ตั้งแต่เริ่ม + commit สม่ำเสมอ และ**ต้องตั้งอีเมลนี้เท่านั้น** ไม่งั้น Vercel จะบล็อกการ deploy:
   ```
   git config user.name "suppo"
   git config user.email "namememem02-ui@users.noreply.github.com"
   ```
6. **GitHub**: บัญชี `namememem02-ui` (gh CLI login ไว้แล้วในเครื่อง) — ก่อนสร้าง repo ให้ถามผู้ใช้ว่า private หรือ public
7. **Deploy**: Vercel ฟรี ผูกกับ GitHub — push แล้ว deploy เอง (~1 นาที) | ห้ามใช้ Node API/fs กับเว็บที่ต้องเป็น static | โปรเจ็คที่ต้องใช้ Java/Python รันได้แค่ในเครื่อง (LAUNCH ชี้ localhost)

## 6. การลงทะเบียนยูนิตใน Hub (อ้างอิงเร็ว)

- แก้ผ่าน UI ที่ localhost:3000 (โหมดหัวหน้ากิลด์) หรือแก้ไฟล์ `Mirai Gallery\data\guilds\<กิลด์>.json` ตรงๆ (เขียนไฟล์ด้วย UTF-8 เสมอ — ระวัง curl ทำภาษาไทยเพี้ยน ให้ใช้ Python เขียน)
- ชื่อกิลด์ภาษาไทยได้ (ระบบ gen id ให้เอง), ยูนิตย้ายกิลด์ได้ด้วยปุ่ม "⇄ ย้ายสังกัด" (สายสัมพันธ์จะถูกปลด)
- อัปเดตเว็บ Hub: `git add -A && git commit && git push` ใน `Mirai Gallery`
