# ออกแบบเครื่องมือแยกหน้า ตัดหน้า และการตั้งชื่อก่อนดาวน์โหลด

วันที่: 2026-07-15

## เป้าหมาย

แยกความหมายของเครื่องมือจัดการหน้า PDF ให้ชัดเจน เพิ่มการแยกทุกหน้าเป็นไฟล์ย่อย และบังคับให้ผู้ใช้ยืนยันหรือแก้ชื่อก่อนดาวน์โหลดไฟล์ทุกครั้งทั้งโปรแกรม โดยยังคงประมวลผลไฟล์ทั้งหมดในเบราว์เซอร์

## ขอบเขต

### ตัดหน้า PDF

- คง route เดิม `/split` และพฤติกรรมเดิมไว้
- เปลี่ยนชื่อที่แสดงจาก “แยกหน้า PDF” เป็น “ตัดหน้า PDF”
- ปรับคำอธิบายและข้อความการทำงานให้ชัดว่า ผู้ใช้เลือกหน้าที่ต้องการเก็บ และหน้าที่ไม่เลือกจะถูกตัดออก
- ผลลัพธ์ยังเป็น PDF หนึ่งไฟล์ที่รวมเฉพาะหน้าที่เลือก

### แยกหน้า PDF

- เพิ่มเครื่องมือใหม่ที่ route `/split-pages`
- เมื่อผู้ใช้วาง PDF ระบบแสดงภาพตัวอย่างทุกหน้าและเลือกทุกหน้าไว้เป็นค่าเริ่มต้น
- ผู้ใช้ยกเลิกหน้าที่ไม่ต้องการส่งออกได้
- แต่ละหน้าที่เลือกถูกสร้างเป็น PDF หน้าเดียว
- ชื่ออัตโนมัติใช้รูปแบบ `<ชื่อเดิม>_หน้า_001.pdf` โดยเติมเลขหน้าสามหลัก
- ผู้ใช้ดาวน์โหลดไฟล์เดี่ยว หรือรวมไฟล์ที่เลือกเป็น ZIP ได้
- เมื่อดาวน์โหลด ZIP ผู้ใช้แก้ได้เฉพาะชื่อ ZIP ส่วนชื่อ PDF ภายในใช้ชื่ออัตโนมัติ

## ระบบตั้งชื่อก่อนดาวน์โหลดส่วนกลาง

- ทุกปุ่มดาวน์โหลดในทุกเครื่องมือ หน้าพรีวิว และหน้าคิวดาวน์โหลด ต้องเปิดโมดัลยืนยันชื่อก่อนเสมอ
- โมดัลแสดงชื่อไฟล์ในช่องแก้ไข และแสดงนามสกุลแยกต่างหากโดยแก้ไม่ได้
- ชื่อเริ่มต้นมาจากชื่อผลลัพธ์ที่เครื่องมือนั้นกำหนดไว้
- ตัดช่องว่างหัวท้าย และแทนอักขระที่ Windows ไม่รองรับ ได้แก่ `\\ / : * ? " < > |`
- ปุ่มดาวน์โหลดใช้ไม่ได้เมื่อชื่อว่าง
- Enter ยืนยันการดาวน์โหลด และ Escape ยกเลิก
- ระบบรักษานามสกุลเดิมให้อัตโนมัติ

### การดาวน์โหลดหลายไฟล์

- การกด “ดาวน์โหลดทั้งหมด” หรือการดาวน์โหลดผลลัพธ์หลายไฟล์ จะรวมไฟล์เป็น ZIP
- ระบบถามชื่อ ZIP เพียงครั้งเดียว
- ชื่อไฟล์ภายใน ZIP ใช้ชื่อเดิมของรายการ และป้องกันชื่อซ้ำโดยเติมลำดับเมื่อจำเป็น

## ส่วนประกอบ

1. โมดัลตั้งชื่อไฟล์ส่วนกลาง รับชื่อเริ่มต้น นามสกุล และ callback สำหรับยืนยันหรือยกเลิก
2. ตัวจัดการดาวน์โหลดส่วนกลาง เปิดโมดัลก่อนเรียก browser download และสร้าง ZIP สำหรับหลายไฟล์
3. เครื่องมือ `/split-pages` สำหรับโหลดภาพตัวอย่าง เลือกหน้า สร้าง PDF หน้าเดียว และส่งผลลัพธ์เข้าระบบดาวน์โหลด
4. หน้า `/split` เดิมที่ปรับเฉพาะชื่อและถ้อยคำให้เป็น “ตัดหน้า PDF”
5. รายการเครื่องมือและเอกสารผู้ใช้ที่อัปเดตชื่อและเพิ่มเมนูใหม่

## ลำดับข้อมูล

1. เครื่องมือสร้าง `Blob` พร้อมชื่อเริ่มต้นและส่งเข้าคิวดาวน์โหลด
2. ผู้ใช้กดดาวน์โหลด ระบบกลางเปิดโมดัลตั้งชื่อ
3. ระบบตรวจและปรับชื่อให้ปลอดภัย
4. ถ้าเป็นไฟล์เดี่ยว ระบบดาวน์โหลด `Blob` ด้วยชื่อที่ยืนยัน
5. ถ้าเป็นหลายไฟล์ ระบบสร้าง ZIP ฝั่งเบราว์เซอร์ เปิดโมดัลชื่อ ZIP แล้วดาวน์โหลด ZIP

## การจัดการข้อผิดพลาด

- ปฏิเสธไฟล์ที่ไม่ใช่ PDF และแจ้งข้อความภาษาไทย
- แจ้งเมื่อ PDF เสียหาย ถูกล็อก หรือเปิดไม่ได้
- ห้ามสร้างผลลัพธ์เมื่อไม่มีหน้าที่เลือก
- ถ้าสร้างไฟล์หน้าใดไม่สำเร็จ ให้ยกเลิกผลลัพธ์ชุดนั้นและแจ้งหมายเลขหน้าที่มีปัญหา ไม่ดาวน์โหลด ZIP ที่ไม่ครบ
- ป้องกันชื่อว่าง อักขระต้องห้าม และชื่อซ้ำภายใน ZIP
- แสดงสถานะความคืบหน้าเมื่อสร้างภาพตัวอย่างหรือ PDF หลายหน้า

## การทดสอบและการตรวจยืนยัน

- ทดสอบตัวปรับชื่อ: ช่องว่าง อักขระต้องห้าม ชื่อว่าง และการรักษานามสกุล
- ทดสอบการสร้างชื่อหน้าสามหลักและการแก้ชื่อซ้ำใน ZIP
- ทดสอบว่าการดาวน์โหลดไฟล์ทุกทางเปิดโมดัลตั้งชื่อก่อนเสมอ
- ทดสอบว่าดาวน์โหลดหลายไฟล์สร้าง ZIP และถามเฉพาะชื่อ ZIP
- ทดสอบว่า `/split-pages` เลือกทุกหน้าเป็นค่าเริ่มต้นและส่งออกเฉพาะหน้าที่เลือก
- ทดสอบว่า `/split` แสดงชื่อ “ตัดหน้า PDF” และยังส่งออกเฉพาะหน้าที่เลือกตามพฤติกรรมเดิม
- รัน lint, TypeScript/Next.js build และทดสอบการใช้งานจริงบนพอร์ต 4200

## นอกขอบเขต

- ไม่อัปโหลดไฟล์ไปยังเซิร์ฟเวอร์
- ไม่เปลี่ยน URL เดิม `/split`
- ไม่เปิดให้แก้ชื่อ PDF แต่ละไฟล์ภายใน ZIP
- ไม่เพิ่มระบบจัดเก็บชื่อไฟล์ข้ามการรีเฟรชหน้าเว็บ
